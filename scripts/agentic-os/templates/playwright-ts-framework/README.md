# Generated test suite — Playwright + TypeScript

A standalone, human-runnable test suite generated by **Agentic OS**. It runs
entirely on its own — no Agentic OS, no Docker, no Java/Maven. (Issue #369.)

## Prerequisites

- **Node.js LTS** (≥ 20). That is the only host prerequisite — `run-tests.sh`
  installs the npm dependencies and the Chromium browser itself.

## Layout

```
.
├── package.json          # @playwright/test + ajv (API schema companions)
├── playwright.config.ts  # reporters (HTML + JUnit), failure artifacts
├── tsconfig.json
├── run-tests.sh          # install deps + browser, then run the suite
└── tests/                # default location; specs live under the SUT's
    ├── api/*.spec.ts      #   configured tests_dir, and the runner discovers
    └── ui/*.spec.ts       #   *.spec.ts anywhere in the bundle
```

API specs (`…/api/*.spec.ts`) use `APIRequestContext` and need no browser; UI
specs (`…/ui/*.spec.ts`) drive Chromium.

## Point it at a SUT

The specs read their target URLs from the environment and **self-skip** when a
prerequisite variable is absent — so set only what your suite needs:

| Env var          | Used by      | Example                          |
| ---------------- | ------------ | -------------------------------- |
| `API_BASE_URL`   | API specs    | `https://sut.example.com/api`    |
| `UI_BASE_URL`    | UI specs     | `https://sut.example.com`        |

Auth/secret env vars (e.g. a bearer token, or `*_OTHER_ID` for BOLA canaries,
or `*_RESPONSE_SCHEMA` for schema validation) are documented in each spec's
header comment. They are read from the environment, never hard-coded.

## Run

```bash
export API_BASE_URL=https://sut.example.com/api
export UI_BASE_URL=https://sut.example.com
./run-tests.sh                 # the whole suite
./run-tests.sh api/            # only API specs (path filter, any tests_dir)
./run-tests.sh --grep @smoke   # tag filter
npm run report                 # open the HTML report
```

On a fresh Linux host the browser also needs system libraries — install once:

```bash
npx playwright install --with-deps chromium
```

## Reports

- `playwright-report/` — HTML report (`npm run report` to open).
- `test-results/results.xml` — JUnit XML for CI.
- Traces / screenshots / video are captured on failure.

Exit code is Playwright's: `0` = all passed, `1` = test failures.
